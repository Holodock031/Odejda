===========================================
  КАК ПРАВИЛЬНО ТЕСТИРОВАТЬ:
===========================================

ВАЖНО! Перед тестированием:
1. Очистите кэш браузера (Ctrl+Shift+Delete)
2. Откройте консоль браузера (F12)
3. Очистите localStorage:
   - Откройте консоль
   - Введите: localStorage.clear()
   - Нажмите Enter

===========================================
  ТЕСТ 1: РЕГИСТРАЦИЯ
===========================================

1. Откройте http://localhost:8080/auth/
2. Переключитесь на "Регистрация"
3. Заполните форму:
   - Имя: Тест Пользователь
   - Email: test999@example.com
   - Телефон: +7 (999) 123-45-67
   - Пароль: test123456
   - Подтверждение: test123456
4. Нажмите "Зарегистрироваться"

ОЖИДАЕМЫЙ РЕЗУЛЬТАТ:
✅ Появится сообщение "Регистрация успешна!"
✅ Перенаправление на главную страницу
✅ В навбаре отображается "Тест" (первое слово имени)
✅ Клик на "Тест" ведет в профиль

===========================================
  ТЕСТ 2: ВЫХОД
===========================================

1. Находясь на главной странице
2. Кликните на имя в навбаре (откроется профиль)
3. Нажмите кнопку "Выйти" в левой панели

ОЖИДАЕМЫЙ РЕЗУЛЬТАТ:
✅ Появится сообщение "Вы вышли из аккаунта"
✅ Перенаправление на главную страницу
✅ В навбаре отображается "Войти"
✅ localStorage очищен (проверьте в консоли: localStorage.getItem('currentUser'))

===========================================
  ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ:
===========================================

1. ОЧИСТИТЕ КЭШ БРАУЗЕРА:
   - Нажмите Ctrl+Shift+Delete
   - Выберите "Кэш" и "Файлы cookie"
   - Нажмите "Очистить"

2. ОЧИСТИТЕ LOCALSTORAGE:
   - Откройте консоль (F12)
   - Введите: localStorage.clear()
   - Обновите страницу (F5)

3. ПЕРЕЗАПУСТИТЕ СЕРВЕР:
   - Остановите сервер (Ctrl+C)
   - Запустите снова: backend\start.bat

4. ПРОВЕРЬТЕ КОНСОЛЬ:
   - Откройте DevTools (F12)
   - Вкладка Console
   - Не должно быть красных ошибок

5. ПРОВЕРЬТЕ NETWORK:
   - Откройте DevTools (F12)
   - Вкладка Network
   - Все запросы должны возвращать 200 или 201
   - Если 403 - проблема с авторизацией
   - Если 404 - проблема с URL

===========================================
  ЛОГИКА РАБОТЫ:
===========================================

РЕГИСТРАЦИЯ/ВХОД:
1. Пользователь заполняет форму
2. Данные отправляются на сервер
3. Django создает сессию
4. Данные пользователя сохраняются в localStorage
5. Навбар обновляется
6. Перенаправление на главную

ВЫХОД:
1. Пользователь нажимает "Выйти"
2. localStorage очищается СРАЗУ
3. Запрос на сервер /api/auth/logout/
4. Django удаляет сессию
5. Навбар обновляется
6. Перенаправление на главную

ПРОФИЛЬ:
1. Страница загружается
2. Проверка авторизации через /api/auth/user/
3. Если НЕ авторизован - перенаправление на /auth/
4. Если авторизован - загрузка данных через /api/profile/
5. Отображение формы профиля

===========================================
  КОМАНДЫ ДЛЯ ОТЛАДКИ:
===========================================

В консоли браузера:

// Проверить localStorage
console.log(localStorage.getItem('currentUser'))

// Очистить localStorage
localStorage.clear()

// Проверить сессию
fetch('/api/auth/user/', {credentials: 'include'})
  .then(r => r.json())
  .then(d => console.log(d))

// Выйти
fetch('/api/auth/logout/', {
  method: 'POST',
  credentials: 'include',
  headers: {'X-CSRFToken': document.cookie.match(/csrftoken=([^;]+)/)?.[1]}
}).then(r => r.json()).then(d => console.log(d))
