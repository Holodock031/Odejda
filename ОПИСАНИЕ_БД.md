# Описание базы данных Fashionista

## Общая информация

**СУБД:** SQLite3  
**Файл БД:** `backend/db.sqlite3`  
**ORM:** Django ORM  
**Кодировка:** UTF-8

---

## Структура базы данных

### 1. Таблица: `auth_user` (Пользователи)

Стандартная таблица Django для пользователей.

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| username | VARCHAR(150) | Имя пользователя (используется email) | UNIQUE, NOT NULL |
| email | VARCHAR(254) | Email пользователя | NOT NULL |
| password | VARCHAR(128) | Хешированный пароль | NOT NULL |
| first_name | VARCHAR(150) | Имя | NULL |
| last_name | VARCHAR(150) | Фамилия | NULL |
| is_staff | BOOLEAN | Является ли сотрудником | DEFAULT FALSE |
| is_active | BOOLEAN | Активен ли аккаунт | DEFAULT TRUE |
| is_superuser | BOOLEAN | Суперпользователь | DEFAULT FALSE |
| date_joined | DATETIME | Дата регистрации | NOT NULL |
| last_login | DATETIME | Последний вход | NULL |

**Индексы:**
- PRIMARY KEY на `id`
- UNIQUE на `username`

**Связи:**
- Один ко многим с `shop_order` (пользователь → заказы)
- Один ко многим с `shop_review` (пользователь → отзывы)
- Один к одному с `shop_userprofile` (пользователь → профиль)

---

### 2. Таблица: `shop_category` (Категории товаров)

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| name | VARCHAR(100) | Название категории | NOT NULL |
| slug | VARCHAR(50) | URL-friendly название | UNIQUE, NOT NULL |
| description | TEXT | Описание категории | NULL |

**Индексы:**
- PRIMARY KEY на `id`
- UNIQUE на `slug`

**Связи:**
- Один ко многим с `shop_product` (категория → товары)

**Примеры данных:**
```sql
INSERT INTO shop_category (name, slug, description) VALUES
('Платья', 'dresses', 'Элегантные платья для любого случая'),
('Верхняя одежда', 'outerwear', 'Пальто, куртки, жакеты'),
('Обувь', 'shoes', 'Туфли, ботинки, кроссовки'),
('Аксессуары', 'accessories', 'Сумки, украшения, шарфы');
```

### 3. Таблица: `shop_product` (Товары)

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| name | VARCHAR(200) | Название товара | NOT NULL |
| description | TEXT | Описание товара | NOT NULL |
| price | DECIMAL(10,2) | Цена в рублях | NOT NULL |
| category_id | INTEGER | ID категории | FOREIGN KEY, NOT NULL |
| image | VARCHAR(100) | Путь к изображению | NULL |
| image_url | VARCHAR(200) | URL изображения | NULL |
| sizes | JSON | Доступные размеры | DEFAULT '[]' |
| colors | JSON | Доступные цвета | DEFAULT '[]' |
| rating | FLOAT | Рейтинг (0-5) | DEFAULT 0 |
| reviews_count | INTEGER | Количество отзывов | DEFAULT 0 |
| in_stock | BOOLEAN | В наличии | DEFAULT TRUE |
| created_at | DATETIME | Дата создания | AUTO_NOW_ADD |
| updated_at | DATETIME | Дата обновления | AUTO_NOW |

**Индексы:**
- PRIMARY KEY на `id`
- INDEX на `category_id`
- INDEX на `in_stock`
- INDEX на `created_at`

**Связи:**
- Многие к одному с `shop_category` (товары → категория)
- Один ко многим с `shop_orderitem` (товар → позиции заказов)
- Один ко многим с `shop_review` (товар → отзывы)

**Пример данных:**
```sql
INSERT INTO shop_product (name, description, price, category_id, image_url, sizes, colors, rating, reviews_count) VALUES
('Элегантное черное платье', 'Идеальное платье для вечернего выхода', 3499.00, 1, 
 'https://example.com/dress.jpg', '["XS","S","M","L"]', '["Черный"]', 4.8, 42);
```

### 4. Таблица: `shop_order` (Заказы)

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| user_id | INTEGER | ID пользователя | FOREIGN KEY, NOT NULL |
| status | VARCHAR(20) | Статус заказа | NOT NULL |
| total | DECIMAL(10,2) | Общая сумма | NOT NULL |
| customer_name | VARCHAR(100) | Имя покупателя | NOT NULL |
| customer_email | VARCHAR(254) | Email покупателя | NOT NULL |
| customer_phone | VARCHAR(20) | Телефон покупателя | NOT NULL |
| delivery_address | TEXT | Адрес доставки | NOT NULL |
| created_at | DATETIME | Дата создания | AUTO_NOW_ADD |
| updated_at | DATETIME | Дата обновления | AUTO_NOW |

**Статусы заказа:**
- `new` - Новый
- `processing` - В обработке
- `shipped` - Отправлен
- `delivered` - Доставлен
- `cancelled` - Отменен

**Индексы:**
- PRIMARY KEY на `id`
- INDEX на `user_id`
- INDEX на `status`
- INDEX на `created_at`

**Связи:**
- Многие к одному с `auth_user` (заказы → пользователь)
- Один ко многим с `shop_orderitem` (заказ → позиции)

---

### 5. Таблица: `shop_orderitem` (Позиции заказа)

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| order_id | INTEGER | ID заказа | FOREIGN KEY, NOT NULL |
| product_id | INTEGER | ID товара | FOREIGN KEY, NOT NULL |
| quantity | INTEGER | Количество | NOT NULL, >= 1 |
| price | DECIMAL(10,2) | Цена за единицу | NOT NULL |
| size | VARCHAR(10) | Размер | NULL |
| color | VARCHAR(50) | Цвет | NULL |

**Индексы:**
- PRIMARY KEY на `id`
- INDEX на `order_id`
- INDEX на `product_id`

**Связи:**
- Многие к одному с `shop_order` (позиции → заказ)
- Многие к одному с `shop_product` (позиции → товар)

**Вычисляемое поле:**
- `total_price` = `price` × `quantity`

---

### 6. Таблица: `shop_review` (Отзывы)

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| product_id | INTEGER | ID товара | FOREIGN KEY, NOT NULL |
| user_id | INTEGER | ID пользователя | FOREIGN KEY, NOT NULL |
| rating | INTEGER | Оценка (1-5) | NOT NULL, 1-5 |
| comment | TEXT | Текст отзыва | NOT NULL |
| created_at | DATETIME | Дата создания | AUTO_NOW_ADD |

**Индексы:**
- PRIMARY KEY на `id`
- INDEX на `product_id`
- INDEX на `user_id`
- UNIQUE на (`product_id`, `user_id`) - один отзыв от пользователя на товар

**Связи:**
- Многие к одному с `shop_product` (отзывы → товар)
- Многие к одному с `auth_user` (отзывы → пользователь)

---

### 7. Таблица: `shop_userprofile` (Профили пользователей)

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Первичный ключ | PRIMARY KEY, AUTO_INCREMENT |
| user_id | INTEGER | ID пользователя | FOREIGN KEY, UNIQUE, NOT NULL |
| phone | VARCHAR(20) | Телефон | NULL |
| address | TEXT | Адрес | NULL |
| avatar | VARCHAR(100) | Путь к аватару | NULL |

**Индексы:**
- PRIMARY KEY на `id`
- UNIQUE на `user_id`

**Связи:**
- Один к одному с `auth_user` (профиль → пользователь)

## SQL запросы для работы с БД

### Получить все товары с категориями

```sql
SELECT 
    p.id,
    p.name,
    p.price,
    c.name as category_name,
    p.rating,
    p.reviews_count
FROM shop_product p
JOIN shop_category c ON p.category_id = c.id
WHERE p.in_stock = 1
ORDER BY p.created_at DESC;
```

### Получить заказы пользователя

```sql
SELECT 
    o.id,
    o.status,
    o.total,
    o.created_at,
    COUNT(oi.id) as items_count
FROM shop_order o
LEFT JOIN shop_orderitem oi ON o.id = oi.order_id
WHERE o.user_id = ?
GROUP BY o.id
ORDER BY o.created_at DESC;
```

### Получить детали заказа

```sql
SELECT 
    oi.id,
    p.name as product_name,
    oi.quantity,
    oi.price,
    oi.size,
    oi.color,
    (oi.quantity * oi.price) as total
FROM shop_orderitem oi
JOIN shop_product p ON oi.product_id = p.id
WHERE oi.order_id = ?;
```

### Получить отзывы товара

```sql
SELECT 
    r.id,
    r.rating,
    r.comment,
    r.created_at,
    u.first_name as user_name
FROM shop_review r
JOIN auth_user u ON r.user_id = u.id
WHERE r.product_id = ?
ORDER BY r.created_at DESC;
```

### Обновить рейтинг товара

```sql
UPDATE shop_product
SET 
    rating = (
        SELECT AVG(rating) 
        FROM shop_review 
        WHERE product_id = ?
    ),
    reviews_count = (
        SELECT COUNT(*) 
        FROM shop_review 
        WHERE product_id = ?
    )
WHERE id = ?;
```

## Миграции Django

Для создания/обновления структуры БД используйте:

```bash
# Создать миграции
python manage.py makemigrations

# Применить миграции
python manage.py migrate

# Откатить миграцию
python manage.py migrate shop 0001

# Показать список миграций
python manage.py showmigrations
```

## Резервное копирование

### Создать бэкап

```bash
# Полный дамп БД
python manage.py dumpdata > backup.json

# Дамп конкретного приложения
python manage.py dumpdata shop > shop_backup.json

# Дамп конкретной модели
python manage.py dumpdata shop.Product > products_backup.json
```

### Восстановить из бэкапа

```bash
python manage.py loaddata backup.json
```

## Оптимизация

### Индексы для производительности

```sql
-- Индекс для поиска товаров по категории
CREATE INDEX idx_product_category ON shop_product(category_id);

-- Индекс для фильтрации товаров в наличии
CREATE INDEX idx_product_in_stock ON shop_product(in_stock);

-- Индекс для сортировки по дате
CREATE INDEX idx_product_created ON shop_product(created_at);

-- Индекс для поиска заказов пользователя
CREATE INDEX idx_order_user ON shop_order(user_id);

-- Индекс для фильтрации по статусу
CREATE INDEX idx_order_status ON shop_order(status);
```

### Рекомендации

1. Используйте `select_related()` для связанных объектов
2. Используйте `prefetch_related()` для обратных связей
3. Добавьте кэширование для часто запрашиваемых данных
4. Используйте пагинацию для больших списков

## Безопасность

1. **Пароли** хешируются с помощью PBKDF2
2. **SQL-инъекции** предотвращаются Django ORM
3. **XSS** защита встроена в Django templates
4. **CSRF** токены для всех POST запросов
