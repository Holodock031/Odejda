**Название:** Fashionista - Интернет-магазин женской одежды

## Страницы сайта

### 1. Главная страница (`/`)

**Файл:** `backend/templates/index.html`  
**View:** `shop.frontend_views.index`  
**JavaScript:** `main.js`, `init.js`

**Содержимое:**
- Герой-секция с баннером
- Категории товаров
- Популярные товары
- О компании
- Контакты

**Функционал:**
- Навигация по сайту
- Счетчик корзины
- Кнопка входа/профиля
- Кнопка админки

---

### 2. Каталог товаров (`/catalog/`)

**Файл:** `backend/templates/catalog.html`  
**View:** `shop.frontend_views.catalog`  
**JavaScript:** `catalog.js`, `main.js`

**Содержимое:**
- Фильтры по категориям
- Поиск товаров
- Сортировка
- Карточки товаров
- Пагинация

**Функционал:**
- Фильтрация по категориям (Все, Платья, Верхняя одежда, Обувь, Аксессуары)
- Поиск по названию
- Сортировка (по цене, названию)
- Добавление в корзину
- Просмотр деталей товара (модальное окно)
- Добавление в избранное

**API endpoints:**
- `GET /api/products/` - список товаров
- `GET /api/products/{id}/` - детали товара
- `GET /api/categories/` - список категорий

**Хранение:**

## API Endpoints

### Товары

```
GET    /api/products/              # Список товаров
GET    /api/products/{id}/         # Детали товара
GET    /api/products/{id}/reviews/ # Отзывы товара
```

**Параметры фильтрации:**
- `category` - фильтр по категории
- `min_price` - минимальная цена
- `max_price` - максимальная цена
- `search` - поиск по названию
- `ordering` - сортировка (price, -price, name, -name)
- `page` - номер страницы

### Категории

```
GET    /api/categories/            # Список категорий
GET    /api/categories/{slug}/     # Детали категории
```

### Заказы

```
GET    /api/orders/                # Список заказов пользователя
POST   /api/orders/                # Создать заказ
GET    /api/orders/{id}/           # Детали заказа
```

### Авторизация

```
POST   /api/auth/register/         # Регистрация
POST   /api/auth/login/            # Вход
POST   /api/auth/logout/           # Выход
GET    /api/auth/user/             # Текущий пользователь
```

### Профиль

```
GET    /api/profile/               # Получить профиль
PUT    /api/profile/               # Обновить профиль
```

### Отзывы

```
GET    /api/reviews/               # Список отзывов
POST   /api/reviews/               # Создать отзыв
GET    /api/reviews/{id}/          # Детали отзыва
PUT    /api/reviews/{id}/          # Обновить отзыв
DELETE /api/reviews/{id}/          # Удалить отзыв
```

## Пользовательские сценарии

### Сценарий 1: Покупка товара

1. Пользователь открывает главную страницу
2. Переходит в каталог
3. Фильтрует товары по категории
4. Просматривает детали товара
5. Добавляет товар в корзину
6. Переходит в корзину
7. Изменяет количество (если нужно)
8. Нажимает "Оформить заказ"
9. Если не авторизован - перенаправление на /auth/
10. Авторизуется или регистрируется
11. Возвращается в корзину
12. Заполняет данные доставки
13. Подтверждает заказ
14. Получает номер заказа

### Сценарий 2: Регистрация

1. Пользователь открывает /auth/
2. Переключается на форму регистрации
3. Заполняет поля:
   - Имя
   - Email
   - Телефон (опционально)
   - Пароль
   - Подтверждение пароля
4. Принимает условия использования
5. Нажимает "Зарегистрироваться"
6. Система создает аккаунт
7. Автоматический вход
8. Перенаправление на главную

### Сценарий 3: Просмотр профиля

1. Пользователь авторизован
2. Кликает на свое имя в навбаре
3. Открывается страница профиля
4. Просматривает личные данные
5. Редактирует телефон/адрес
6. Сохраняет изменения
7. Просматривает историю заказов
8. Может выйти из аккаунта

## Технические детали

### Авторизация

**Механизм:**
- Django Session Authentication
- CSRF токены для безопасности
- localStorage для кэширования данных пользователя

**Процесс входа:**
1. Пользователь вводит email и пароль
2. POST запрос на `/api/auth/login/`
3. Django создает сессию
4. Возвращаются данные пользователя
5. Данные сохраняются в localStorage
6. Обновляется навбар

**Процесс выхода:**
1. Пользователь нажимает "Выйти"
2. localStorage очищается
3. POST запрос на `/api/auth/logout/`
4. Django удаляет сессию
5. Перенаправление на главную

### Корзина

**Хранение:**
- localStorage (ключ: `cart`)
- Не требует авторизации для просмотра
- Требует авторизации для оформления заказа

**Синхронизация:**
- Счетчик обновляется при каждом изменении
- Данные сохраняются автоматически
- При оформлении заказа корзина очищается

### Стили

**CSS Framework:** Vanilla CSS (без фреймворков)  
**Размер:** 36KB  
**Особенности:**
- CSS переменные для цветов
- Адаптивный дизайн (mobile-first)
- Медиа-запросы: 992px, 768px, 576px, 400px
- Анимации и переходы
- Flexbox и Grid

**Основные цвета:**
```css
--primary: #e83e8c;        /* Розовый */
```

## Безопасность

### Реализованные меры

1. **CSRF защита** - токены для всех POST запросов
2. **XSS защита** - экранирование в Django templates
3. **SQL инъекции** - Django ORM
4. **Хеширование паролей** - PBKDF2
5. **Session security** - HttpOnly cookies
6. **CORS настройки** - разрешены только локальные запросы

### Рекомендации для продакшена

1. Изменить `SECRET_KEY` в settings.py
2. Установить `DEBUG = False`
3. Настроить `ALLOWED_HOSTS`
4. Использовать HTTPS
5. Настроить CORS правильно
6. Использовать PostgreSQL вместо SQLite
7. Настроить логирование
8. Добавить rate limiting

## Развертывание

### Локальная разработка

```bash
# 1. Активировать виртуальное окружение
cd backend
.\venv\Scripts\activate

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Применить миграции
python manage.py migrate

# 4. Загрузить тестовые данные
python manage.py load_initial_data

# 5. Создать суперпользователя (опционально)
python manage.py createsuperuser

# 6. Собрать статические файлы
python manage.py collectstatic

# 7. Запустить сервер
python manage.py runserver 8080
```

### Продакшен

```bash
# 1. Установить gunicorn
pip install gunicorn

# 2. Собрать статику
python manage.py collectstatic --noinput

# 3. Запустить с gunicorn
gunicorn fashionista.wsgi:application --bind 0.0.0.0:8080
```
